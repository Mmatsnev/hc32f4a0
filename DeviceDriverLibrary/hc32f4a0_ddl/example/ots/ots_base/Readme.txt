平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程展示了OTS的基本用法，中断用法以及用如何外部触发源触发OTS。

说明：
1. OTS可选用内置的默认参数，也可使用通过定标实验获得的参数，通过‘APP_OTS_USE_DEFAULT_PARA’
   控制。‘APP_OTS_USE_DEFAULT_PARA’为非0时，选用内置的默认参数，不需要定标实验；
   ‘APP_OTS_USE_DEFAULT_PARA’为0时，必须先通过定标实验获得参数，才能使用OTS进行测温，每颗
   MCU都必须做定标实验。
2. 使用外部触发源触发OTS工作，本例程采用timer2每秒触发一次OTS；
3. 例程当前的配置为使用内置的默认参数，使用外部触发源触发OTS工作。

================================================================================
使用步骤
================================================================================
1）如果需要用在上位机观察温度值，请将USB转串口工具正确地与开发板上的J28端口（靠近
   Micro USB接口）连接；如果不需要，请忽略此步骤；
2）打开工程并重新编译；
3）启动IDE的下载和调试功能；
4）用设置断点的方式查看温度值，或用USB转串口结合串口调试助手软件，在上位机观察温度值；
5）修改宏‘APP_OTS_USE_TRIG’的定义为非0，可使用外部触发源来触发OTS测温————每秒测量一次。

平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程以ADC的单元3（M4_ADC3）的序列A为例，用序列A单次扫描模式，实现了以轮询方式对指定的
ADC通道对应的引脚进行采样。这些通道对应的引脚包括一个与电位器相连的引脚（ADC123_IN3，默
认与ADC通道3对应），例程用串口将该引脚的ADC值输出。例程关于ADC的配置包括：ADC时钟配置，
ADC初始化，ADC通道配置和ADC平均通道配置。

说明：
1. 宏“APP_PRIMARY_CLK”指定ADC的时钟源；
2. 宏“APP_ADC_UNIT”指定需要使用的ADC单元，还可定义为ADC1或ADC2；
3. 宏“APP_ADC_SA_CH”指定需要使用的ADC通道，ADC1和ADC2最大支持16个通道，ADC3最大支持20
   个通道；
4. 宏“APP_ADC_SA_SAMPLE_TIME”指定序列A各通道的采样时间，按通道顺序设置；ADC3的通道16~
   通道19共享同一个采样时间；
5. 宏“APP_ADC_AVG_CH”指定需要进行多次采样计算平均值的通道，如果不需要，将其定义为“0”即可；
6. 宏“APP_ADC_VREF”，是引脚VREFH的电压，开发板不同，其值可能略有差异。

================================================================================
使用步骤
================================================================================
1）如果需要用在上位机观察ADC值，请将USB转串口工具正确地与开发板上的J28端口（靠近Micro
   USB接口）连接；如果不需要，请忽略此步骤；
2）打开工程并重新编译；
3）启动IDE的下载和调试功能；
4）用设置断点的方式查看ADC值，或用USB转串口结合串口调试助手软件，在上位机观察ADC值；
5）旋转电位器，可观察到ADC值的变化。

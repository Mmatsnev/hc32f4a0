平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程以ADC的单元3（M4_ADC3）的序列A为例，用序列A单次扫描模式，实现了以轮询方式
对指定的ADC通道对应的引脚进行采样。这些通道对应的引脚包括一个与电位器相连的引脚
（ADC123_IN3，默认与ADC通道3对应），例程用串口（波特率115200）将该引脚的ADC值输
出。例程关于ADC的配置包括：ADC时钟配置，ADC初始化，ADC通道配置和ADC平均通道配置。

================================================================================
使用步骤
================================================================================
1）将电位器旁的J13开关拨为VEXT<->VR；
2）如果需要用在上位机观察ADC值，请将USB转串口工具正确地与开发板上的J28端口（靠近
   Micro USB接口）连接；如果不需要，请忽略此步骤；
3）打开工程并重新编译；
4）启动IDE的下载和调试功能；
5）用设置断点的方式查看ADC值，或用USB转串口结合串口调试助手软件，在上位机观察ADC值；
6）旋转电位器，可观察到ADC值的变化。

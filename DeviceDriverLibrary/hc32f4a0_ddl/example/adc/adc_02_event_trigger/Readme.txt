平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程以ADC的单元1（M4_ADC1）的序列A序列B为例，用序列A序列B单次扫描模式，实现了用事件
和引脚ADTRGx来触发ADC扫描。例程用事件EVT_TMR2_1_CMPA（由Timer2的通道A产生）来触发序列
A扫描，用引脚PE7的下降沿来触发序列B扫描。例程还描述了ADC中断的配置和基本用法。

说明：
序列A和序列B不能包含相同的通道。
序列B的优先级高于序列A，序列B会中断序列A的扫描。结构体“stc_adc_init_t”中的“u16SAResumePos”
用于指定，序列A被序列B中断后，恢复扫描的位置，可指定为从序列A的第一个通道恢复扫描，也可指
定为从被中断的通道恢复扫描。

================================================================================
使用步骤
================================================================================
1）如果需要用在上位机观察程序运行情况，请将USB转串口工具正确地与开发板上的J28端口（靠近
   MicroUSB接口）连接；如果不需要，请忽略此步骤；
2）打开工程并重新编译；
3）启动IDE的下载和调试功能；
4）全速运行程序或在需要的位置设置断点；
5）序列A由Timer2每秒触发一次扫描；序列B由PE7的下降沿触发，在PE7上制造一个下降沿，即可触发
   序列B扫描。

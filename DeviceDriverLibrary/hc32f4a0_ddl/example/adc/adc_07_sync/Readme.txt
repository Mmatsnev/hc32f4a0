平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程使用了全部的ADC单元来演示ADC的协同模式，并将各ADC的扫描时序通过TIMERA的PWM输出，
可通过示波器观察。

说明：
工作在协同模式的ADC单元：
1. 只使用序列A，禁止序列B；
2. 禁止同一引脚在不同单元同时被采样；
3. 应使用相同的初始化配置；
4. 序列A使能相同的通道数；单元之间，序列A使能的通道可以不相同，但通道间的相对位置应相同。
   如：ADC1使能通道1000010100000000，ADC2可使能通道0000000010000101或0000001000010100等
5. 各通道的采样时间应相同。
6. 例程中当ADC和ADC2协同工作(APP_SYNC_UNITS == ADC_SYNC_ADC1_ADC2)，且协同模式为循环
   延迟触发模式时(APP_SYNC_MODE == APP_SYNC_MODE_CONT_SEQUENTIAL)，只配使能了一个模拟
   通道，且其对应的是同一个外部模拟输入引脚，这样配置的目的是为了实现对同一个模拟输入引脚
   以5Mbps的速率进行采样。

================================================================================
使用步骤
================================================================================
1）将电位器旁的J13开关拨为VEXT<->VR；
2）打开工程并重新编译；
3）启动IDE的下载和调试功能；
4）在PE7上制造一个下降沿，触发各ADC单元扫描，用示波器观察各单元扫描时序。

平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。


功能描述
================================================================================
本样例主要展示MCU的TIMER6的硬件启动、停止、清零功能
说明：
配置timer6，timer6计数时钟设置为240M/128，周期约为533nS
PI6 -->PWMA 
PI5 -->PWMB

硬件启动: PI6 -->PWMA 上升沿

硬件停止: PI6 -->PWMA 下降沿

硬件清零: PI5 -->PWMB 上升沿

使用步骤
================================================================================
1） PI6连接PB9(PWMA)，PI5连接PB8(PWMB)
2） 打开工程编译，分别在代码以下行设置断点：
    GPIO_SetPins(TEST_IO_A_PORT, TEST_IO_A_PIN); /* Hardware start count */
    GPIO_ResetPins(TEST_IO_A_PORT, TEST_IO_A_PIN); /* Hardware stop count */
    GPIO_SetPins(TEST_IO_B_PORT, TEST_IO_B_PIN); /* Hardware clear count register */
    GPIO_ResetPins(TEST_IO_B_PORT, TEST_IO_B_PIN); /* Hardware clear count register */
3） 运行代码，观察寄存器TMR61的值
    可以观察到，当PWMA检测到上升沿时，Timer6计数启动，CNTER寄存器的值产生变化；
    当PWMA检测到下降沿时，Timer6计数停止，CNTER寄存器的值不再变化；
    当PWMB检测到上升沿时，Timer6 CNTER寄存器的值被清零；

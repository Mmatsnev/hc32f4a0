平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。


功能描述
================================================================================
本样例主要展示MCU的HRPWM外设对TIMER6的PWM输出的校正功能，计算了两个HRPWM单元的单
位延迟量。并通过HRPWM的CH1和CH5为例演示了对PWM占空比的微调功能。

说明：
配置timer6，计数时钟为240M,所有单元的PWMA,PWMB都同步输出相同的PWM波形,高电平时间
约33.3nS。

         Timer6      HRPWM channel
Timer61：PWMA（PB9)  CH1
         PWMB（PB8） CH2
Timer62：PWMA（PB7） CH3
         PWMB（PB14）CH4
Timer63：PWMA（PA10）CH5
         PWMB（PB15）CH6
Timer64：PWMA（PE14）CH7
         PWMB（PB10）CH8
Timer65：PWMA（PC10）CH9
         PWMB（PD0） CH10
Timer66：PWMA（PA2） CH11
         PWMB（PA3） CH12
Timer67：PWMA（PA4） CH13
         PWMB（PA5） CH14
Timer68：PWMA（PC2） CH15
         PWMB（PC3） CH16

如样例代码中宏定义EXAM_HRPWM_FUNC_ON被定义，则使能了CH1和CH5的HRPWM功能，配置如下：
CH1的上升沿向后延迟1个单位延迟量；
     下降沿向后延迟1个单位延迟量；
CH5的上升沿向后延迟CALCODE/2个单位延迟量；
     下降沿向后延迟CALCODE/2个单位延迟量。

使用步骤
================================================================================
1） 打开工程编译并全速运行。
2） 停止工程运行，查看变量fPrecision0和fPrecision1计算出的单位延迟量（单位：nS）。
    通过计算可以得出以下理论值：
    CH1的上升沿延迟时间 = CH1的下降沿延迟时间 = 1 x fPrecision0
    CH5的上升沿延迟时间 = CH5的下降沿延迟时间 = T（PCLK0）/2 约2.08nS
    
3） 使用示波器查看CH1，CH5 的PWM波形，
    可以看到，CH5的上升沿比CH1上升沿延迟约为(2.08nS - CH1的上升沿延迟时间);
    CH5的下降沿比CH1下降沿延迟约为(2.08nS - CH1的下降沿延迟时间);
